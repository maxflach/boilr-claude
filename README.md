# Claude Project System

One-command full-stack project scaffolding with Claude Code. Spin up a production-ready TypeScript project in minutes, then build features with a team of specialized AI agents.

---

## What This Is

Two Claude Code skills that work together:

- **`/initproject`** — Run once in an empty directory. Scaffolds the whole project, installs deps, creates a git repo on GitHub, generates an AI agent team, and writes the `/newfeature` skill into the project.
- **`/newfeature`** — Run inside a project to build a feature end-to-end. A Project Manager plans it, specialist agents build it phase by phase, a Code Reviewer checks it.

Full documentation: **[`claude-project-system.md`](./claude-project-system.md)**

---

## Requirements

- [Claude Code](https://claude.ai/claude-code) — installed and authenticated
- **Node.js** v20+ and **npm**
- **git**
- **GitHub CLI** — `brew install gh` then `gh auth login`
- **Firebase CLI** — `npm install -g firebase-tools` then `firebase login`

## Installation

### 1. Copy the skill file

```bash
mkdir -p ~/.claude/skills/initproject
cp skills/initproject/SKILL.md ~/.claude/skills/initproject/SKILL.md
```

### 2. Verify

Open a new Claude Code session anywhere and type `/initproject` — if it appears in autocomplete, you're set.

## Using `/initproject`

```bash
mkdir ~/code/my-app && cd ~/code/my-app
claude
/initproject
```

You'll be asked:
1. **Stack** — Firebase (Firestore + Functions) or Node.js + React (PostgreSQL + Prisma + Express)
2. **App name**
3. **Multi-tenant?**
4. **Auth provider** — Firebase Auth, JWT, or None
5. **GitHub visibility** — private or public

Each major phase pauses for your approval before continuing.

At the end you'll have a full scaffold, git repo on GitHub, agent team, and the `/newfeature` skill ready to go.

---

## Using `/newfeature`

```bash
cd ~/code/my-app
claude
/newfeature
```

1. Describe the feature, set priority, flag DB changes
2. The **Project Manager** (Claude Opus) writes a detailed plan
3. You review and approve (or adjust) the plan
4. A `feature/<slug>` branch is created from `develop`
5. Specialist agents build it phase by phase — DB → Backend → Frontend → DevOps
6. Each phase pauses for your review
7. **Code Reviewer** checks everything at the end

## Stack Options

### Firebase Stack
Firestore + Firebase Functions + Firebase Auth + Firebase Hosting

Best for: real-time apps, rapid prototyping, serverless.

### Node.js + React Stack
PostgreSQL + Prisma + Express + React

- Frontend → Firebase Hosting
- API → Firebase App Hosting (managed Node.js)
- OpenAPI docs auto-generated at `/api/doc`
- Frontend client auto-generated by hey-api (no raw fetch calls)

Best for: relational data, complex queries, REST APIs.

## The Agent Team

Every project gets a team of Claude agents, each with a defined scope:

| Agent | Model | Role |
|---|---|---|
| Project Manager | Opus | Plans features, orchestrates agents, never writes code |
| Frontend Engineer | Sonnet | React components, hooks, pages |
| Backend Engineer | Sonnet | API routes, business logic |
| Database Engineer | Opus | Schema changes, migrations, queries |
| DevOps | Sonnet | Config, deployment, CI/CD |
| Code Reviewer | Opus | Reviews all changes, blocks `useEffect` abuse |

## The Rules (baked into every project)

### No `useEffect` for data fetching — ever

```tsx
// Never — causes race conditions and memory leaks
useEffect(() => { fetchUser(id).then(setUser); }, [id]);

// Always — use React Query
const { data: user } = useQuery({ queryKey: ['user', id], queryFn: () => getUser(id) });
```

### Pass IDs as props, not full objects

```tsx
// No
<OrderRow order={order} />

// Yes — React Query deduplicates, so this is free
<OrderRow orderId={order.id} />
```

### No prop drilling beyond 1 level

Use a React Query hook (server data) or Jotai atom (UI state) instead.

### Small components — one responsibility, ~150 lines max

## What's in `boilrapi/`

The Node.js + React stack uses **BoilrApi** — a lightweight Express wrapper that gives you:

- TypeBox schema validation (compiled once at startup, fast)
- JWT auth with role checks
- Auto-generated OpenAPI spec + Swagger UI at `/api/doc`
- File uploads via multer
- OAuth via Passport

The `boilrapi/` directory contains all the TypeScript source files. The `initproject` skill copies these directly into your project — no npm package required.

See **[`claude-project-system.md`](./claude-project-system.md)** for the full API architecture docs.
